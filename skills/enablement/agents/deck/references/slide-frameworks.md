# Enablement Deck Slide Frameworks

Detailed guidance for each slide type used in Wati CSA enablement decks.

---

## The Enablement Narrative Arc

Unlike a sales deck (problem → product → close), an enablement deck assumes the customer has already bought. The arc is:

**Why this matters → What it is → How to set it up → How to use it well → What good looks like → What to do next**

The goal: the audience leaves able to DO something with this feature they couldn't do before.

---

## Slide Type Reference

### Cover Slide
- Feature name: 40pt bold, centered
- Tagline: 20pt, one line, captures the key benefit (not a feature description)
- Date + session type in small text bottom left
- Dark background with Wati teal accent

**Good tagline examples:**
- "Resolve support queries before your agents even wake up" (AI Support Agent)
- "Unlock the Power of Voice" (WhatsApp Calling)
- "Turn WhatsApp conversations into conversion machines" (Broadcasts)

**Bad tagline examples:**
- "Introducing AI Support Agent" (not a benefit)
- "Everything you need to know about Broadcasts" (too vague)

---

### Agenda Slide
- Title: "What We'll Cover Today"
- 4–5 items as a numbered or icon list
- Each item = 3–5 words (not full sentences)
- Light background, clean layout

---

### Problem / Why This Matters Slide
- Lead with a stat or a pain statement — not with Wati
- 1–2 numbers that quantify the problem
- Visual: striking large number or simple illustration
- Keep to 1 core problem — don't scatter

**Structure:**
```
[Large stat or bold pain statement]
[2-line explanation of what this means for them]
[Supporting stat or second proof point]
```

**Example (WhatsApp Calling deck):**
```
77%
believe a phone call is the fastest way to get an answer

In a digital-first world, the human connection is 
your competitive advantage.

40% [supporting stat]
```

---

### Feature Introduction Slide
- Title: "Introducing [Feature Name]"
- One-sentence definition (no jargon)
- 3–4 capability bullets — outcome-focused, not feature-named
- Product screenshot if available: right side of slide or full-width below text

**Formula for each capability bullet:**
> [What it does]: [Why it matters in one phrase]

Example:
> One-Click Call from Chat: Enable customers to start a voice call without switching apps

---

### Setup Step Slides
- Title: "Step [N]: [Action]" or "[Step Name]"
- Screenshot fills 60–70% of slide
- Annotate the screenshot: circle or arrow pointing to the key UI element
- 2–3 bullet instructions alongside the screenshot
- Keep each slide to ONE action — don't combine steps

---

### Feature Deep Dive Slides
- Title: the capability name
- Left column: what it does + why it matters (3–4 bullets)
- Right column: screenshot
- Bottom: optional stat or outcome callout

If no screenshot: use an icon grid or a "before/after" two-column layout.

---

### Best Practices Slides
- Title: "[Feature] Best Practices" or "Getting the Most from [Feature]"
- 3 practices per slide, each with:
  - Bold short title (4–6 words)
  - 1–2 sentence explanation
  - Specific, actionable guidance (numbers, settings, thresholds)

**Good best practice:**
> **Set Confidence Threshold to 70–80%**
> This balances accuracy with coverage. Too high = too many escalations. Too low = wrong answers.

**Bad best practice:**
> **Configure your settings correctly**
> Make sure everything is set up properly for best results.

---

### Case Study / Success Story Slide
Structure:
```
[Customer name + role + company]
[Challenge: 1 sentence]
[Solution: 1 sentence — what they did with Wati]
[Result: 1–2 metrics — before → after]
[Quote: pull quote in large italic text]
```

Always use a real Wati customer when possible. Reference `.agents/wati-brand.md` for the customer list.

---

### Resources Slide
- Title: "Resources"
- 3 items max:
  1. Support knowledge base: support.wati.io
  2. Feature-specific setup guide (link from brief if provided)
  3. Best practices doc or video (if available)
- Clean icon + link format, no long URLs

---

### Next Steps / CTA Slide
- Title: "Next Steps" or "You're Ready — Here's What to Do"
- 3 actions, numbered:
  1. Immediate action (set up the feature today)
  2. Support path (raise a ticket / support.wati.io)
  3. 1-to-1 path (book a call with your CSM)
- One prominent CTA button or QR code if available

---

## Anti-Patterns to Avoid

**The feature dump**: 10 slides listing every sub-feature. Nobody remembers it. Pick the 4 most impactful.

**Setup slides without screenshots**: If you're telling someone where to click, show them where to click.

**Best practices without specifics**: "Configure properly" is not a best practice. Give the exact setting, threshold, or approach.

**Skipping the problem slide**: If the audience doesn't feel the pain first, they won't care about the solution. Always start with why.

**Ending without a clear next step**: The last action slide should have exactly ONE thing to do. Not three options, not "let us know." One specific step.

**Generic customer stories**: "A company in retail saw improvement." Name them. Give the number. Quote them.
