# Wati Launch Day — Quarterly Product Updates Webinar Pattern

Extracted from Wati's real quarterly decks:
- **Q1 2026 — "What's Cooking at Wati?" (Wati X Astra)** — 38 slides (preferred format)
- **Q4 '25 — "What's New at Wati"** — 43 slides

"Wati Launch Day" is Wati's quarterly customer-facing webinar where all the features shipped that quarter are introduced to customers. Audience = external customers across all plans.

---

## The Skeleton (follow this order)

### 1. Title slide
- Big: `[Q# Year]` + a hook line ("What's Cooking at Wati?" / "What's New at Wati")
- Sub-hook (optional): a one-line theme for the quarter ("The Complete Customer Journey — Powered by AI")
- Footer line: `Product Updates  |  Online  |  [Month Year]`

### 2. Quarter at a Glance
Three oversized stats + a one-line summary of what the quarter was about.
```
[One-line quarter narrative — what this quarter was really about]
40+ Features Shipped    100+ Bugs Quashed    8+ Channels Now Live
```
Pull real numbers from the release log. If a number isn't known, mark `[# — TBC]`.

### 3. Themes That Define the Quarter
The whole quarter organized into **4–6 numbered themes**. This is the spine of the deck.

**Theme ordering rule (important):** The lead theme is **the PMM's call — always ask/confirm which theme leads.** Do not assume a fixed leader. The default reasoning to propose: lead with the **biggest launch / biggest MRR impact** of the quarter (what the audience remembers), then order the rest by impact. The PMM may override for strategic reasons. (Trend for reference: Q1 led with AI at Wati; Q4 led with Channels/Customer Journey; Q2 2026 the PMM chose Astra to lead.) When unsure, propose an order and let the user confirm before building.
```
01  AI at Wati            — Voice agents, Copilot insights, agents that build themselves
02  Multichannel          — WhatsApp, Instagram, Messenger, RCS, SMS, TikTok — start anywhere
03  Sales & Marketing     — Campaigns that fire themselves, segments that stay fresh
04  Ecommerce Growth      — Full-funnel Shopify automation with multichannel fallback
```
Each theme = a two-digit number + short name + one-line description.

### 4. Per-Theme Block (repeat for each theme)

**a. Theme divider slide**
```
#N
[Theme Name]
[One-line description of what this theme covers]
```

**b. Feature slides** — one per major feature. Use the THREE-BLOCK format:
```
[Feature name as a benefit headline — e.g. "Voice 2.0: Clone Any Voice in < 1 Min"]

THE PROBLEM
[The pain in the customer's world — 1-2 sentences. Concrete, not abstract.]

THE UPDATE
[What shipped — the mechanic, plainly. 1-2 sentences.]

WHAT'S IN IT FOR YOU
[The outcome/benefit — why they should care. 1 sentence.]

Plan: [Growth / Pro / Business / All Plans / Astra Plans / Add-on]
```

**c. Demo slides** — deliberately minimal, placed right after the feature they demo:
```
Demo: [Feature Name]
```
(A play affordance + Storylane/loom link placeholder. Nothing else.)

**d. "Also Shipped in [Quarter]" grid** — minor features that don't need a full slide. 3–6 items, each: bold feature name + one-line description. Closes the theme.

### 5. Bonus — Pro Tips
A "Bonus / Pro Tips" divider, then 1–2 tip slides. Standard recurring tips:
- **Pro Tip #1: Try a Higher Plan** — no-commitment "Try Higher Plan" program, upgrade icon top of dashboard
- **Pro Tip #2: Get Astra AI Agents** — 30-day free trial, User Management → Build with Astra

### 6. What's Coming Next Quarter
A roadmap sneak-peek grid. 4–6 upcoming items, each: bold name + one-line teaser. Frames the next Launch Day.
```
What's Coming in [Next Quarter]
A sneak peek at the product roadmap for [months]
[Feature] — [one-line teaser]  ×4–6
```

### 7. Q&A
```
Q and A
Questions? Reach out via the Wati widget in your dashboard.
Feedback on today's session? Drop a note at pmm@wati.io
```

### 8. Thank You
Clean close with `www.wati.io  |  pmm@wati.io`.

## Non-Negotiable Format Signatures

- **Footer on every content slide:** `www.wati.io  |  pmm@wati.io` + page number (bottom).
- **Plan tag on every feature slide** — customers must know eligibility. Never omit.
- **THE PROBLEM / THE UPDATE / WHAT'S IN IT FOR YOU** — this three-block rhythm is the heart of the Q1 format. Use it for every major feature.
- **Numbered themes (#1, #2…)** with divider slides — the organizing system.
- **Demo slides are minimal** — just "Demo: [name]". Never cram content onto them.
- **"Also Shipped" grids** absorb the long tail of minor features so the deck stays scannable.
- **Roadmap teaser** always closes the content before Q&A — it builds anticipation for next quarter.

---

## Copy Voice (from the real decks)

- Feature headlines are benefit-led and specific: "Campaigns Launch the Moment Meta Approves", "Start Wati Without a WhatsApp Number" — not "Campaign Scheduling Update".
- THE PROBLEM is written in the customer's voice, concrete: "A blank agent configuration screen is where good intentions go to die."
- WHAT'S IN IT FOR YOU is one punchy outcome line: "Sales reps call instead of qualify."
- No fabricated stats or customer quotes — mark `[STAT — TBC]`.
- Plain English. No "we're excited to announce."

---

## Inputs This Skill Needs

To build a Launch Day deck, gather:
1. **The quarter's shipped features** — from the release log / product team / Notion. For each: name, the problem it solves, what it does, the benefit, and plan eligibility.
2. **Quarter stats** — features shipped, bugs fixed, channels live (for the "at a glance" slide).
3. **Theme grouping** — how the features cluster into 4–6 themes (AI, Multichannel, Sales & Marketing, Ecommerce are the recurring ones).
4. **Next-quarter roadmap** — 4–6 upcoming items for the teaser.
5. **Screenshots / demo links** — one per feature that gets a demo slide.

If features aren't yet grouped into themes, propose a grouping and confirm with the user before building.
