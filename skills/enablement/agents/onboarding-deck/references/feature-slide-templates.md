# Onboarding-Deck Feature-Slide Templates

Catalog of the feature-slide templates used in Wati's marketing/onboarding decks (`Wati for Marketing_ Onboarding Deck.pptx` and `Wati for Marketing_ Deck Draft_ Final.pptx`). When a feature launches, pick the template whose shape matches the feature's size/importance, then CLONE that real slide and swap the content.

All templates follow the Wati design system (`context/design-system.md`): DM Sans, brand palette, burst on headings, logo top-right, footer.

## How to choose a template (by feature size/importance)
- **Small / simple feature** → **T1 (Text + single mockup)** or **T5 (3-up)** — one slide.
- **Medium feature** (a few capabilities) → **T2 (Text + visual grid)** or **T5 (3-up icon rows)** — one slide.
- **Big / flagship feature** (marquee, high MRR) → **2–3 slides**: a **T0 section header** + **T1/T2 detail** (+ **T6 outcomes/proof**). Do it justice.
- **Feature that beats an alternative / has a "vs"** → **T4 (two-column / comparison)**.
- **Feature with a flow/process** (tracking, multi-step) → **T3 (diagram/flow)**.
Confirm slide count with Sunny — he may ask for 1, or 2–3 for a big feature.

## Templates

### T0 — Journey / Section header
Tinted background, stage/theme name, short line, a row of labeled icons (e.g. Awareness & Acquisition, Engagement, Conversion, Retention). Use to open a big-feature cluster.

### T1 — Text-left + phone-mockup-right  *(the workhorse feature slide)*
Left: feature headline + 3 benefit bullets (bold sub-head + one line each). Right: a phone/product screenshot mockup. Best default for a single feature with one clear UI. (Ref: "Ads that click to WhatsApp + Remarketing".)

### T2 — Text-left + visual-grid-right
Left: headline + short intro. Right: a grid of small visuals/sub-features (catalogue, menus, cards). Use when a feature has several UI elements to show. (Ref: "Engage customers with rich interactive messages".)

### T3 — Diagram / flow
Headline + a horizontal/step diagram of a process, with short labels. Use for tracking/multi-step mechanics. (Ref: "Conversion API (CAPI) — the Template Message Approach".)

### T4 — Two-column / comparison
Two side-by-side panels: with/without, AI+human, option A/B. Use when the feature's value is a contrast. (Ref: "AI-agents + human agents"; "Vs other BSPs / cPaaS".)

### T5 — 3-up icon rows
Three (or four) benefit blocks, each = icon + bold head + one line. Use for a feature whose story is 3 crisp benefits. (Ref: chatbots capabilities; WhatsApp Flows use cases.)

### T6 — Case study / stat callout
Big stat callouts (💸 / 📞 / 📊 style) + a customer quote + name/role. Use to prove a big feature with a real customer. Only real stats/quotes — else omit. (Ref: Habuild / Bowaba / Blacklyf slides.)

## Content each feature slide needs (pull from `context/*.md` + support article)
- Feature name (exact, per naming rules in `context/positioning.md`)
- One-line "what it does for the customer" (benefit-led)
- 3 key benefit points (for T1/T5) OR the visual/flow (T2/T3)
- Plan eligibility
- What to pitch it *against* (the pain / alternative it beats) — the onboarding deck is a sales-pitch aid
- A real proof point if the feature is big (T6)
